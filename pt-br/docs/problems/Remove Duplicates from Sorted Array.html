26. Remove Duplicates from Sorted Array

https://leetcode.com/problems/remove-duplicates-from-sorted-array/description

Implementação​

class Solution:
    def removeDuplicates(self, nums: list[int]) -> int:
        left_index, right_index = 0, 0
        n = len(nums)

        while right_index < n:
            left, right = nums[left_index], nums[right_index]

            if left != right:
                left_index += 1
                nums[left_index] = right
            
            right_index += 1
        
        return left_index + 1

Testes​

import pytest
from .solution import Solution

@pytest.mark.parametrize('numbers,expected', [
    ([1, 1, 1, 2, 2, 3, 3], [1, 2, 3])
])
def test_solution(numbers, expected):
    new_size = Solution().removeDuplicates(numbers)
    assert numbers[:new_size] == expected

Intuição​

A ideia é encontrar o próximo elemento L[j] que seja diferente do elemento atual L[i]. Uma vez que a lista L já está ordernada, é garantido que L[j] >= L[i]. Dessa forma, quando L[i] != L[j], aplicamos as seguintes operações:

L[i+1]=L[j]L[i + 1] = L[j]L[i+1]=L[j]
i=i+1i = i + 1i=i+1
j=j+1j = j + 1j=j+1

Este processo se repete enquanto j < N, isto é, enquanto j é um índice válido.

8ª iteração​

Como j = 7 e 7 < 7 == false, o algoritmo termina e retorna o tamanho da lista resultante, que nesse caso é i + 1 == 3. Dessa forma a solução fica:

1 2 3 2 2 3 3

Complexidade de tempo​

Como a lista é iterada por completo da esquerda para a direita através do ponteiro j, a complexidade é:

O(n)O(n)O(n)

Complexidade de espaço​

Como a lista é modificada in-place e nenhuma alocação além de variáveis auxiliares é feita, a complexidade é:

O(1)O(1)O(1)
